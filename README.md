# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

### Интерфейсы

#### Интерфейс IProduct
Интерфейс описывает сущность Товар. Его поля описывают характеристики товаров из каталога интернет-магазина. 

Поля интерфейса:
`id: string` - описывает поле, которое хранит уникальный идентификатор товара.  
`description: string` - описывает поле, которое хранит подробное описание товара.
`image: string` - описывает поле, которое хранит изображение товара.
`title: string` - описывает поле, которое хранит название товара.
`category: string` - описывает поле, которое хранит категорию товара.
`price: number | null` - описывает поле, которое хранит цену товара. Может содержать значение null, в случае когда нет возможности оценить товар.

#### Интерфейс IBuyer
Интерфейс описывает сущность Покупатель. Его поля описывают данные покупателя, необходимые для оформления заказа.

Поля интерфейса:
`payment: TPayment | null` - описывает поле, которое хранит возможные способы оплаты в магазине. Может содержать три значения - наличная оплата, безналичная оплата или null, если покупатель еще не определился, какой способ оплаты будет использован.
`email: string` - описывает поле, которое хранит адрес электронной почты покупателя.
`phone: string` - описывает поле, которое хранит номер телефона покупателя. 
`address: string` - описывает поле, которое хранит адрес покупателя.
 
### Модели данных

#### Класс Catalog
Класс предназначен для работы со всем ассортиментом товаров, имеющемся в интернет-магазине.

Конструктор класса не принимает параметров.

Поля класса:
`product: IProduct[]` - хранит весь массив товаров, имеющийся в каталоге.
`selectedProduct: IProduct` - хранит карточку товара, выбранную для добавления в корзину.

Методы класса:
`getProduct(): IProduct[]` - получает весь массив товаров из модели. 
`setProduct(product: IProduct[]): void` - сохраняет массив товаров, полученный в параметрах метода, в поле product класса.
`getSelectedProduct(): IProduct | null` - получает информацию о выбранном товаре для подробного отображения карточки товара. Возвращает информацию о товаре, либо null, если товар не выбран.
`setSelectedProduct(selectedProduct: IProduct): void` - сохраняет информацию о выбранном товаре в поле selectedProduct класса.
`getProductById(id: string): IProduct | undefined` - получает информацию по конкретному товару по его id, полученному в параметрах метода. Если товар не найден, то метод возвращает undefined.

#### Класс Cart
Класс предназначен для работы с данными о товарах, отобранными покупателем в корзину.

Поля класса:
`productCart: IProduct[] | null` - хранит массив товаров, отобранных покупателем в корзину. Может содержать значение null, если корзина пустая.

Конструктор класса не принимает параметров.

Методы класса:
`getProductCart(): IProduct[] | null` - получает массив товаров из корзины. Может содержать null, если корзина пустая.
`setProductCart(product: IProduct): void` - добавляет товар, полученный в параметре функции, в корзину.
`delProductCart(product: IProduct): void` - удаляет товар, полученный в параметре функции, из корзины.
`clearCart(): void` - удаляет все товары из корзины.
`getTotalPrice(): number` - получает общую стоимость всех товаров в корзине.
`getCountProduct(): number` - получает количество товаров в корзине.
`checkProductCartById(id: string): boolean` - проверяет наличие товара в корзине по его id.

#### Класс Buyer
Класс предназначен для работы с данными покупателя.

Конструктор класса не принимает параметров.

Поля класса:
`payment: TPayment | null` - хранит тип оплаты, выбранный покупателем, может содержать null, если покупатель еще не определился, какой тип оплаты использовать.
`address: string` - хранит почтовый адрес покупателя.
`email: string` - хранит электронный адрес покупателя.
`phone: string` - хранит номер телефона покупателя.

Методы класса:
`getDataBuyer(): IBuyer` - получает все данные по покупателю, оформляющему заказ.
`checkDataBuyer(): TCheckError` - проверяет данные о покупателе на валидность.
`clearDataBuyer(): void` - очищает все данные о покупателе.
`setDataBuyer(dataBuyer: { payment?: TPayment, address?: string, email?: string, phone?: string }): void` - добавляет данные по одному или нескольким полям в информацию о покупателе.

## Слой коммуникации

### Класс WebApi

Класс предназначен для выполнения запросов на сервер и получения с сервера объекта с массивом товаров.

Конструктор:
`constructor(api: IApi)` - принимает объект, соответствующий интерфейсу IApi, для выполнения сетевых запросов.

Поля класса:
`api: IApi` - хранит экземпляр класса для работы с запросами.

Методы класса:
`getProductList(): Promise<IProductList>` - выполняет GET-запрос на эндпоинт /product/ и возвращает объект, содержащий массив товаров.
`orderProduct(order: IOrder): Promise<IOrderResult>` — выполняет POST-запрос на эндпоинт /order, передавая данные о заказе и покупателе, и возвращает объект, подтверждающий покупку на определенную сумму.

## Слой представления (View)

### Класс Header

Класс отвечает за отображение данных в блоке header интернет-магазина. Наследуется от базового класса Component.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает в качестве параметра весь контейнер header. В конструкторе происходит поиск элементов кнопки и счетчика с последующим присваиванием элементов полям buttonBasket и counterBasket соответственно.

Поля класса:
`buttonBasket: HTMLButtonElement` - поле, которое отвечает за отображение элемента кнопка корзины.
`counterElement: HTMLElement` - поле, которое отвечает за отображение счетчика товаров, добавленных в корзину.

Методы класса:

`counter (value: number): void` - сеттер, предназначенный для изменения счетчика товаров в корзине.

### Класс Gallery

Класс отвечает за отображение каталога товаров интернет-магазина. Наследуется от базового класса Component.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает в качестве параметра контейнер, в котором будет размещен каталог товаров.

Поля класса:
`catalogElement: HTMLElement` - элемент, который отвечает за отображение каталога товаров.

Методы класса:
`catalog(items: HTMLElement[]): void` - сеттер, который изменяет каталог товаров на странице магазина.

### Класс Modal
Класс предназначен для работы с модальными окнами. Наследуется от базового класса Component.  

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает в качестве параметра контейнер, в котором размещены два элемента - кнопка и контент для модального окна, а также объект событий.

Поля класса:

`buttonClose:HTMLButtonElement` - элемент кнопки закрытия модального окна.
`contentElement:HTMLElement` - контейнер модального окна, в который динамически вставляется содержимое в зависимости от событий, таких как  нажатие на форму, корзину или генерация информационного сообщения.

Методы класса:

`content(content: HTMLElement): void` - сеттер для установки содержимого модального окна в поле contentElement. 

`openModal(): void` - открывает модальное окно.
`closeModal(): void` - закрывает модальное окно.

### Класс Success
Класс предназначен для вывода в модальное окно сообщения об успешном завершении заказа. Наследуется от базового класса Component.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер шаблона,в котором находятся элементы описания и кнопка подтверждения, а также объект событий.

Поля класса:

`buttonSuccess: HTMLButtonElement` - кнопка подтверждения покупки.
`descriptionElement: HTMLElement` - элемент для вывода текста с итоговой суммой.

Методы класса:
`totalPrice(valuePrice: number)` - сеттер для установки итоговой суммы покупки.

### Класс Card
Базовый класс для всех классов, которые отвечают за отображение карточки товара. В свою очередь наследуется от класса Component.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер элементов и объект событий.

Поля класса:
`titleElement: HTMLElement` - элемент, который отвечает за отображение названия товара и является общим для всех вариантов контейнера карточки.
`priceElement: HTMLElement` - элемент, который отвечает за отображение цены товара и является общим для всех вариантов контейнера карточки.

Методы класса:
`title(strTitle: string): void` - сеттер, который устанавливает название товара в поле titleElement.
`price(valuePrice: number | null): void` - сеттер, который устанавливает цену товара в поле priceElement. Аргумент может содержать null, если цена товара имеет метку "Бесценно".


### Класс CardCatalog
Класс отвечает за отображение карточки товара в каталоге на главной странице. Наследуется от базового класса Card.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - в конструкторе, помимо элементов родителя, дополнительно инициализируются элементы, содержащие информацию о категории товара и его изображение.

Поля класса:
`categoryElement: HTMLElement` - элемент, который отвечает за отображение категории товара. 
`imageElement: HTMLImageElement` - элемент, который отвечает за отображение изображения товара.

Методы класса:
`category(strCategory: string): void` -  сеттер, который устанавливает категорию товара в поле categoryElement.
`image(strImageLink: string): void` - сеттер, который устанавливает ссылку на изображение товара в поле imageElement.

### Класс CardPreview
Класс отвечает за отображение данных с детальным описанием товара в отдельном модальном окне. Наследуется от базового класса Card.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` -  в конструкторе, помимо элементов родителя, дополнительно инициализируются элементы, содержащие информацию о категории товара, развернутое описание, изображение товара, а также элемент кнопки добавления в корзину.

Поля класса:
`categoryElement: HTMLElement` - элемент, который отвечает за отображение категории товара. 
`imageElement: HTMLImageElement` - элемент, который отвечает за отображение изображения товара.
`descriptionElement: HTMLElement` - элемент, который отвечает за отображение подробной информации о товаре.
`buttonElement: HTMLButtonElement` - элемент, который отвечает за отображение кнопки добавления товара в корзину.

Методы класса:
`category(strCategory: string): void` -  сеттер, который устанавливает категорию товара в поле categoryElement.
`image(strImageLink: string): void` - сеттер, который устанавливает ссылку на изображение товара в поле imageElement.
`description(strDescription: string): void` - сеттер, который устанавливает подробное описание товара в поле descriptionElement.
`buttonText(strText: string)` - меняет текст на кнопке buttonElement.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - в конструкторе, помимо элементов родителя, дополнительно инициализируются элементы, содержащие информацию о порядковом номере элемента и кнопка удаления товара из корзины. 

Поля класса:

`buttonDelete: HTMLButtonElement` - элемент, который отвечает за отображение кнопки удаления товара из корзины.

Методы класса:
`index(valueIndex: number): void` - сеттер, который устанавливает индекс в поле indexElement.

### Класс Basket
Класс отвечает за отображение содержимого корзины в виде списка товаров и общей стоимости в отдельном модальном окне. Наследуется от базового класса Component.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - в конструкторе инициализируются элементы списка, итоговой стоимости и кнопка оформления заказа. 

Поля класса:
`listBasket: HTMLElement` - элемент списка, в который будут вставляться карточки товара.
`totalPrice: HTMLElement` - элемент для вывода итоговой стоимости товаров.
`buttonElement: HTMLButtonElement` — элемент для отображения  кнопки перехода к оформлению заказа.

Методы класса:
`list(items: HTMLElement[]): void` - сеттер для заполнения списка товаров в корзине.
`total(valueTotalPrice: number): void` - сеттер для установки итоговой стоимости в элемент totalPrice.

### Класс Form
Базовый класс для всех классов, которые отвечают за работу с формами. В свою очередь наследуется от класса Component.

Конструктор:
`constructor(container:HTMLFormElement, events: IEvents)` - в конструкторе инициализируется кнопка отправки и элемент для вывода ошибок.

Поля класса:
`buttonSubmit: HTMLButtonElement` - элемент кнопки отправки формы.
`errorsForm: HTMLElement` - элемент для вывода сообщений об ошибках валидации.

Методы класса:

## Описание событий, которые генерируются в приложении

`card:select` - выбор карточки.
`card:remove` - удаление карточки из корзины.
`buyer:changed` - изменение данных в buyer.
`formErrors:changed` - вывод результатов валидации.

## Презентер
Презентер не выделяется в отдельный класс, в связи с небольшим объемом проекта, а реализуется в файле main.ts.

В проектной работе Презентер выступает в роли посредника между моделями данных (Model) и их представлениями на экране пользователя (View).

В коде Презентер только обрабатывает события.
